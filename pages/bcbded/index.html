# 生命周期函数

所有的生命周期钩子自动绑定 this 上下文到实例中，因此可以访问数据，对属性和方法进行运算。这意味着不能使用箭头函数来定义一个生命周期方法 (例如 created: () => this.fetchTodos()) 。这是因为箭头函数绑定了父上下文，因此 this 与你期待的组件实例不同，this.fetchTodos 的行为未定义

# beforeCreate

在实例初始化之后，数据观测 data observer 和 event/watcher 事件配置之前被调用

# created

在实例创建完成后被立即调用

  • 实例已完成以下的配置
    • 数据观测 (data observer)
    • property 和方法的运算
    • watch/event 事件回调
  • 然而，挂载阶段还没开始，$el property 目前尚不可用

# beforeMount

在挂载开始之前被调用：相关的 render 函数首次被调用

  • 该钩子在服务器端渲染期间不被调用

# mounted

实例被挂载后调用

  • 此时 Vue.createApp({}).mount() 被新创建的 vm.$el 替换

    • 如果根实例挂载到了一个文档内的元素上，当 mounted 被调用时 vm.$el 也在文档内

  • mounted 不会保证所有的子组件也都一起被挂载

    • 如果希望等到整个视图都渲染完毕，可以在 mounted 内部使用 vm.$nextTick (opens new window)

# beforeUpdate

数据更新时调用，发生在虚拟 DOM 打补丁之前

  • 这里适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器

  • 该钩子在服务器端渲染期间不被调用，因为只有初次渲染会在服务端进行

# updated

由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子

  • 当这个钩子被调用时，组件 DOM 已经更新，可以执行依赖于 DOM 的操作

  • 大多数情况下，应该避免在此期间更改状态。如果要相应状态改变，通常最好使用计算属性 (opens new window)或侦听器 (opens new window)取而代之

  • updated 不会保证所有的子组件也都一起被重绘

  • 如果希望等到整个视图都重绘完毕，可以在 updated 里使用 vm.$nextTick (opens new window)

  • 该钩子在服务器端渲染期间不被调用

# activated

被 keep-alive 缓存的组件激活时调用

  • 该钩子在服务器端渲染期间不被调用

# deactivated

被 keep-alive 缓存的组件停用时调用

  • 该钩子在服务器端渲染期间不被调用

# beforeUnmount

在卸载组件实例之前调用

  • 在这个阶段，实例仍然是完全正常的

  • 该钩子在服务器端渲染期间不被调用

# unmounted

卸载组件实例后调用

  • 调用此钩子时
    • 组件实例的所有指令都被解除绑定
    • 所有事件侦听器都被移除
    • 所有子组件实例被卸载
  • 该钩子在服务器端渲染期间不被调用

# renderTracked

跟踪虚拟 DOM 重新渲染时调用

  • 钩子接收 debugger event 作为参数
    • 此事件说明了是什么操作跟踪了组件以及该操作的目标对象和键

# renderTriggered

当虚拟 DOM 重新渲染为 triggered.Similarly 为renderTracked (opens new window)

  • 接收 debugger event 作为参数
    • 此事件说明了是什么操作触发了重新渲染，以及该操作的目标对象和键

# errorCaptured

当捕获一个来自子孙组件的错误时被调用

  • 此钩子会收到三个参数

    • 错误对象
    • 发生错误的组件实例
    • 一个包含错误来源信息的字符串

  • 此钩子可以返回 false 以阻止该错误继续向上传播

# Nono-props

  • 当父组件向子组件传值时，子组件通过 props 接受
    • 若子组件未接收，则改值会添加到子组件的属性上
    • 若子组件未接收，并且不想在子组件上显示，则可在子组件上设置 ¡Attrs: false
      • 当子组件有多个根结点时，属性不会在子组件上生效，若想生效则可在子组件节点上添加绑定 <div v-bind="$attrs"></div>
      • 若只想父组件传递的某个属性生效，则需子组件节点上绑定属性 <div :msg="$attrs.msg"></div>
      • 未接收时可以用 this.$attrs.msg 来获取属性值

# 父组件向子组件传值

父组件通过属性的方式向子组件传值

子组件可以使用父组件传递的数据，但绝对不能修改

const app = Vue.createApp({
  data() {
    return {
      num: 100
    }
  },
  template: `<div><hello-world :num="num"></hello-world></div>`
})
app.component('hello-world', {
  props: {
    num: {
      type: Number, // 也可以是是多种类型 [Number, String],
      required: true, // 是否必填
      // 设置默认值，基本类型可以直接写，对象或数组必须是一个工厂函数
      default: 1000, 
      /*
      default: function(){
      	return {message: 'hello world'}
      }
      */
      
      // 自定义校验规则，验证条不通过会在控制台产生警告 [Vue warn]: Invalid prop
      validator: function (value) {
        return value > 500
      }
    }
  },
  template: `<div>{{num}}</div>`
})
const vm = app.mount('#root')

# 子组件向父组件传值

子组件通过触发事件携带参数的方式向父组件传值

const app = Vue.createApp({
  data() {
    return {
      num: 1
    }
  },
  template: `<div><Counter @add="handleAdd" :num="num" /></div>`,
  methods: {
    handleAdd: function (step) {
      this.num += step
    }
  }
})
app.component('Counter', {
  props: ['num'],
  // emits 用于提示此组件向外触发了哪些事件，如果触发了 emits 数组以外的事件，控制台会显示警告
  // emits: ['add'],
  // emits 也可以是一个对象，属性名为触发事件名称，属性值为对触发事件携带参数的校验函数
  emits: {
    add: (count) => {
      return count > 0
    }
  },
  template: `<div @click="handleClick">{{num}}</div>`,
  methods: {
    handleClick: function () {
      this.$emit('add', 2)
    }
  }
})
const vm = app.mount('#root')

# Provide / Inject

跨越多层组件传递数据，传递的数据并非是响应的

const app = Vue.createApp({
  data() {
    return {
      name: 'Ethan',
      age: 26
    }
  },
  // 方式一：不使用 data 中的数据，如果使用 data 中的数据不会有效果
  provide: {
    name: 'Ethan',
    age: 26 
  },

  // 方式二：使用 data 中的数据
  provide() {
    return {
      name: this.name,
      age: this.age
    }
  },

  template: `
  	<child-one />
  `
})
app.component('child-one', {
  template: `<child-two />`
})
app.component('child-two', {
  template: `<child-three />`
})
app.component('child-three', {
  inject: ['name', 'age'],
  template: `<div>{{name}}：{{age}}</div>`
})
const vm = app.mount('#root')

# 动态组件

根据数据的变化，结合 component 标签，动态的切换组件

动态组件经常结合 <keep-alive> 标签来缓存切换之前的状态和数据

const app = Vue.createApp({
  data() {
    return {
      currentItem: 'item-input'
    }
  },
  template: `
    <div><button @click="handleClick">切换组件</button></div>
    <keep-alive>
    	<component :is="currentItem" />
    </keep-alive>
  `,
  methods: {
    handleClick: function () {
      this.currentItem =
        this.currentItem === 'item-input' ? 'item-text' : 'item-input'
    }
  }
})
app.component('item-input', {
  template: `
  	<input />
  `
})
app.component('item-text', {
  template: `
  	<div>Hello world</div>
  `
})
const vm = app.mount('#root')

# 小知识点

子组件中 name 的作用

  • 递归组件
  • 对某个页面取消缓存
  • 控制台开发工具中显示的组件名

有些标签内必须嵌套固定的标签，比如 <select> 内部只能是 <option>，<tbody> 中只能是 <tr>，否则会出现显示异常

  • 解决方法：使用 :is 说明调用的组件
const app = Vue.createApp({
  template: `
    <table>
      <tbody>
          <tr v-is="'row'"></tr>
          <tr v-is="'row'"></tr>
          <tr v-is="'row'"></tr>
      </tbody>
    </table>
  `
})
app.component('row', {
  template: '<tr><td>this is a row</td></tr>'
})
const vm = app.mount('#root')

# Mixin

  • 混入 data：组件 data 优先级高于 mixin data 优先级
  • 混入 Methods：组件的 method 高于 mixin method 优先级
  • 混入生命周期函数：先执行 Mixin 的生命周期函数，再执行组件自身的生命周期函数

修改某个属性对于组件内和 Mixin 的使用优先级

app.config.optionMergeStragies.number = (mixinValue, appValue) => {
  return mixinValue || appValue
}

代码示例

<div id="app">
  <div @click="handleClick">{{number}}</div>
  <div>{{count}}</div>
</div>

<script>
  // 局部混入
  const myMixin = {
    data() {
      return {
        number: 2,
        count: 6
      }
    },
    mounted() {
      console.log('myMixin mounted')
    },
    methods: {
      handleClick: function () {
        console.log('myMixin handleClick')
      }
    }
  }
  const app = Vue.createApp({
    el: '#app',
    mixins: [myMixin],
    data() {
      return {
        number: 1
      }
    },
    mounted() {
      console.log('mounted')
    },
    methods: {
      handleClick: function () {
        console.log('handleClick')
      }
    }
  })
  
  // 全局混入
  app.mixin({
    data(){
      return {
        number: 1
      }
    },
    mounted(){
      console.log('mixin mounted')
    }
  })
  
  const vm = app.mount('#app')
</script>

        transition: 'all .4s ease'

        top: 0,

app.directive('pos', {
  mounted(el, binding) {
    el.style.top = binding.value.top + 'px'
    el.style.left = binding.value.left + 'px'
  },
  updated(el, binding) {
    el.style.top = binding.value.top + 'px'
    el.style.left = binding.value.left + 'px'
  }
})

 * 方式二：简写形式
 * 当自定义指令内部只有 mounted 和 update 两个生命周期函数，且操作内容一致时，可以简写

app.directive('pos', (el, binding) => {
  el.style.top = binding.value.top + 'px'

})

 * binding.arg   接收指令名 top
 * binding.value 接收指令的值

app.directive('pos', (el, binding) => {

})

# toRefs

解构被包装的对象，使其子元素也变为响应式引用

const app = Vue.createApp({
  template: `<div>name: {{name}}，age: {{age}}</div>`,
  setup(props, context) {
    const { reactive, toRefs } = Vue
    let nameObj = reactive({ name: 'sunzhenyang', age: 26 })
    setTimeout(() => {
      nameObj.name = 'Ethan'
      nameObj.age = 28
    }, 2000)
    
    /**
     * const { name, age } = nameObj
     * 直接解构出的内部元素 name、age 并不是响应式的，因为只有 nameObj 是被 proxy 包装的，而内部元素并没有被处理

     *    name: proxy({ value: 'sunzhenyang' }
     *    age: proxy({ value: 26 }
     */
    const { name, age } = toRefs(nameObj)
    return {
      name,
      age
    }
  }
})

# watch

具备一定的惰性：首次加载不会执行，值改变时才会执行，可以通过 immediate: true将其变为非惰性

  template: `

      <div>
        <span>Name：</span>
        <input v-model="name" />

      <div>Name：{{name}}</div>
    </div>
   `
  ,
  setup(props, context) {
    const { ref, watch } = Vue
    const name = ref('')
    watch(name, (newValue, oldValue) => {
      console.log(newValue, oldValue)
    },{
      // 变为非惰性
      immediate: true
    })
    return {
      name
    }
  }
})

    • 依赖变动，内部函数会执行。依赖没有变化，内部代码不会执行
  • 无法获取之前数据变化前的值，只能拿到当前的值

  template: `
    <div>
      <div>

      <div>

        <input v-model="name2" />
      </div>

    </div>

  ,

    const { reactive, watchEffect, toRefs } = Vue
    const nameObj = reactive({ name1: 'a', name2: 'b' })

      name1,
      name2
    }
  }
})

const app = Vue.createApp({

    return {
      name
    }
  }
})

const inputRalativeEffect = () => {

# 路由守卫

import { createRouter, createWebHashHistory } from 'vue-router'
import Home from '../views/home/Home'
import Login from '../views/login/Login'
import Register from '../views/register/Register'
const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/register',
    name: 'Register',
    component: Register,
    // 如果进入 Register 路由之前页面已经登录，则跳转回首页
    beforeEnter(to, from, next) {
      const { isLogin } = localStorage
      isLogin ? next({ name: 'Home' }) : next()
    }
  },
  {
    path: '/login',
    name: 'Login',
    component: Login,
    // 如果进入 Login 路由之前页面已经登录，则跳转回首页
    beforeEnter(to, from, next) {
      const { isLogin } = localStorage
      isLogin ? next({ name: 'Home' }) : next()
    }
  }
]
const router = createRouter({
  history: createWebHashHistory(),
  routes
})
router.beforeEach((to, from, next) => {
  const { isLogin } = localStorage
  const { name } = to
  const isLoginOrRegister = name === 'Login' || name === 'Register'
  // 已经登录，或者要前往的是登录或者注册页面的路由，则前往路由所在的页面
  // 未登录：前往登录页面
  isLogin || isLoginOrRegister ? next() : next({ name: 'Login' })
  // 已经登录(注册)再次进入登录(注册)页面时，要跳转到首页，在上方 Login(Register) 路由定义时使用 beforeEnter 进行操作
})
export default router

# VueX

数据管理框架

  • VueX创建了一个全局唯一的仓库，用来存放全局的数据
    • state：状态，保存全局的变量
    • action：行动，调用 同步/异步函数，通过 commit 提交到 mutations
    • mutations：改变状态
  • mutations 中不允许放异步代码，需要异步操作时要把异步操作放在 action 中

# 修改数据

/**
 * 第一种方式：当页面没有异步代码时
 * 直接提交一个 commit，触发一个名为 change 的 mutation
 */
// 页面
this.$store.commit('change')
// store/index.js
export default createStore({
  state: {
    name: 'Ethan'
  },
  mutations: {
    change() {
      this.state.name = 'sunzhenyang'
    }
  },
  actions: {
  },
  modules: {
  }
})

/**
 * 第二种方式：当页面存在异步代码时
 * 	 第一步：调用 dispatch 方法，派发一个名为 change 的 action
 *   第二步：store 感知到一个名为 change 的 action，执行 action 中的 change 方法
 *   第三步：action 中的 change 方法提交一个 commit，触发一个名为 change 的 mutation
 *   第四步：mutation 中对应的 change 方法被执行
 *   第五步：mutation 中的 change 方法中修改数据
 */
// 页面中
// 第一步
this.$store.dispatch("change", 'sunzhenyang');
// store/index.js
export default createStore({
  state: {
    name: 'Ethan'
  },
  mutations: {
    // 第四步
    change(state, str) {
      // 第五步
      state.name = str
    }
  },
  actions: {
    // 第二步
    change(store, str) {
      // 第三步
      setTimeout(() => {        
        store.commit('change', str)
      }, 2000);
    }
  },
  modules: {
  }
})
